# 토스 미니앱(앱인토스) 출시 완전 가이드

먹픽(Meokpick) 프로젝트 진행 경험 기반. 새 앱 출시 시 이 가이드 그대로 따라가면 됨.

---

## 📋 전체 흐름 한눈에 보기

```
[1] 앱 등록 (콘솔)         → 영업일 2일 검토
[2] 사업자 등록 (홈택스)    → 2~3일 발급
[3] 사업자 정보 토스 등록   → 1~2일 검토
[4] 토스 로그인 + 약관 등록 → 즉시
[5] 코드 개발 + 배포        → 작업 시간 따라
[6] .ait 빌드 + 업로드     → 콘솔 빌드 10분
[7] 테스트                  → 직접
[8] 검토 요청              → 영업일 2~3일
[9] 출시                    → 즉시
```

**총 소요 기간: 빠르면 1~2주, 사업자 없으면 처음부터 2~3주**

---

## 1️⃣ 사전 준비 (필수)

### 필요한 것
- ✅ 사업자등록증 (개인사업자, **일반과세자**)
  - 토스 미니앱은 간이과세자 불가
  - 업종코드: `722000` (소프트웨어 개발 및 공급업)
- ✅ 약관 호스팅용 URL (HTML 파일 + Vercel 등)
- ✅ GitHub 계정 (Private 저장소)
- ✅ Vercel 계정 (Hobby 무료)
- ✅ Node.js 18+ 환경

### 사업자 등록 시 주의
- **간이과세자로 신청 시도 → 거부됨** (홈택스에서 전화 옴)
- 처음부터 **일반과세자** 선택
- 통신판매업: 처음엔 "아니오" (광고 수익만이면 불필요, 결제 도입 시 추가)
- 부모님 집 사용 시: 무상사용승낙서는 선택 (없어도 통과되는 경우 많음)
- 처리 기간: 2~3 영업일

## 2️⃣ 앱인토스 콘솔 등록

### 접속: https://apps.toss.im

### 입력 정보
- 앱 이름 (한글)
- 영어 앱 이름 (Display name)
- **appName** (영문, 소문자, 공백 없이) ⭐ 매우 중요
  - `intoss://{appName}` 딥링크에 사용됨
  - 빌드 설정과 일치해야 함
- 카테고리
- 검색 키워드 (5개)
- 고객 문의 이메일
- 사용 연령: **만 19세 이상** (토스 강제 - 변경 불가)

### 디자인 자산 업로드
- **썸네일**: 최대 1920×828 PNG
- **로고**: 정사각형 권장 (600×600 등)
- **스크린샷**: 세로형 636×1048, 최소 3장

### 첫 검토 요청 → 영업일 2일

## 3️⃣ 사업자 정보 등록

### 위치: 앱인토스 콘솔 → 사업자 정보 등록하기

### 필요 자료
- 사업자등록번호 (홈택스 발급)
- 사업자등록증명 PDF (홈택스에서 발급, 3개월 이내)
  - 홈택스 → My홈택스 → 민원증명 → 사업자등록증명
  - 발급 후 PDF 저장

### 입력
- 사업자 등록번호
- 신청하는 사람: 대표자
- 간이과세자 여부: **체크 안 함**
- PDF 첨부

### 검토 → 1~2일

---

## 4️⃣ 토스 로그인 + 약관 등록

### 위치: 콘솔 좌측 메뉴 → 토스 로그인

### 동의 항목 설정
- **이름**: 필수 동의 (변경 불가)
- 나머지(이메일, 성별, 생년월일 등): **사용 안함**
  - 정말 필요한 경우만 받기 (정책상 최소 수집 원칙)

### 약관 등록
- **서비스 이용약관 URL** (필수)
- 마케팅 정보 수신 동의: 광고 메시지 보낼 거면 등록, 아니면 SKIP

### 연결 끊기 콜백
- 이름만 받으면: **미설정 OK**
- 추가 정보 받으면: 콜백 URL 필수

### ⚠️ 약관 URL 형태
- 로컬 파일 경로(`C:\...\terms.md`) 불가
- **반드시 공개 URL** (예: `https://yourapp.vercel.app/terms.html`)

## 5️⃣ 약관 HTML 페이지 만들기

### Vercel에 배포할 경우
`public/terms.html`, `public/privacy.html` 파일로 생성

### 약관 필수 포함 사항
```
- 회사명
- 사업자등록번호
- 대표자
- 주소
- 이메일
- 시행일자
```

### 약관 본문 핵심 조항
**서비스 이용약관**: 목적 / 용어 정의 / 약관 효력 변경 / 서비스 내용 / 이용 / 변경 중단 / 이용자 의무 / 면책 조항 / 지식재산권 / 개인정보 보호 / 분쟁 해결

**개인정보처리방침**: 수집 항목 및 방법 / 처리 목적 / 보유 기간 / 제3자 제공 / 처리 위탁 / 이용자 권리 / 쿠키 / 보호책임자

→ 먹픽 프로젝트의 `docs/PRIVACY_POLICY.md`, `docs/TERMS_OF_SERVICE.md` 참고

## 6️⃣ 코드 개발

### 추천 스택
- **Vite + React** (가장 빠름, 가장 단순)
- TypeScript는 선택 (학습 곡선 vs 안정성)
- localStorage로 간단한 상태 관리 (서버 불필요한 경우)
- React Router 없이 `useState` 라우팅 가능 (페이지 적으면)

### Toss UX 가이드 필수 준수
1. **NavBar**: 좌측 뒤로가기(<), 가운데 제목, 우측 메뉴
2. **줌 비활성화**: `<meta name="viewport" content="..., maximum-scale=1.0, user-scalable=no" />`
3. **alert/confirm 사용 금지** → 커스텀 Modal 제작
4. **언어**: `<html lang="ko">`
5. **테마 색상**: 앱 메인 컬러로 설정
6. **OG 메타 태그**: 카카오톡/SNS 공유 시 미리보기

### 반응형 처리
- PC 미리보기용 고정 사이즈 wrapper 쓰지 말 것
- 처음부터 `100vh / 100vw` 기반
- 토스 WebView는 디바이스 화면 그대로 사용

### 권장 폴더 구조
```
src/
├── App.jsx              # 페이지 라우팅
├── pages/               # 각 페이지
├── components/          # 공통 UI (NavBar, Modal 등)
├── services/            # 비즈니스 로직
└── data/                # 상수, 데이터
```

## 7️⃣ GitHub + Vercel 배포

### GitHub
```sh
git init
git remote add origin https://github.com/<user>/<repo>.git
git push -u origin main
```

**Private 저장소** 권장 (아이디어 보호)

### Vercel
- GitHub OAuth 로그인
- Hobby Plan (무료) 선택
- 저장소 자동 연결 → push 시 자동 배포

### Vercel CLI 설치 시 주의
- 컴퓨터 hostname 한글이면 (`삼성` 등) `vercel login` 에러 발생
- 해결: 브라우저로 OAuth 로그인

## 8️⃣ Apps in Toss Framework 설정

### 설치
```sh
npm install @apps-in-toss/web-framework
```

### granite.config.ts 작성
```ts
import { defineConfig } from '@apps-in-toss/web-framework/config';

export default defineConfig({
  appName: 'yourappname',  // ⭐ 콘솔에 등록한 appName과 정확히 일치
  brand: {
    displayName: '한글 앱 이름',
    primaryColor: '#FF6A00',
    icon: 'https://yourapp.vercel.app/favicon.svg',
  },
  web: {
    host: 'localhost',
    port: 5173,
    commands: {
      dev: 'vite dev',
      build: 'vite build',
    },
  },
  permissions: [],
  outdir: 'dist',
});
```

### .ait 파일 빌드
```sh
npx ait build
```

→ 프로젝트 루트에 `<appName>.ait` 파일 생성됨

## 9️⃣ 토스 콘솔 업로드 + 검토

### 위치: 콘솔 → 앱 출시 → [+ 버전 등록]

1. `<appName>.ait` 파일 첨부
2. 메모: `v1.0 초기 출시 버전`
3. 등록하기 → 토스 측 빌드 10분
4. 빌드 완료 → 상태: **검토 필요**

### 테스트
- 콘솔의 **[테스트]** 버튼 클릭
- QR 코드 또는 `intoss-private://` 링크
- **휴대폰 토스 앱**에서만 열림 (PC 브라우저 X)

### 검토 요청
- **[검토 요청]** 클릭
- **출시 노트** 입력 (사용자에게 보이는 앱 소개)
- **앱 내 기능** 최소 1개 등록
  - 한국어 기능명 (다른 앱과 중복 X)
  - 영어 기능명 (띄어쓰기 제외 15자 이하)
  - 이동 URL: `intoss://{appName}/screenName` (screenName 필수 입력)
- 검토 → 영업일 2~3일

## 🔟 출시

### 검토 통과 후
- 콘솔 → 앱 출시 → 해당 버전 **[출시하기]** 클릭
- 출시 노트 최종 확인
- "모든 사용자에게 출시" 확인

### 출시 후
- **검색 인덱싱**: 1~6시간 ~ 24시간
- 토스 콘솔 **대시보드** → DAU, 사용자 통계 확인 가능
- **이벤트 추적**: 토스 자동 수집 (별도 설정 없이)

## ⚠️ 흔한 함정 / 주의사항

### 1. 사업자 관련
- 간이과세자 선택 → 거부됨, **일반과세자**로
- 업종코드 잘못 선택 → 정정 필요
- 처음 검색되는 `642005`(유선통신업)는 잘못된 코드

### 2. 토스 로그인 권한
- 필요 없는 정보까지 받기 X
- 이름 외 정보 받으면 콜백 URL 필수

### 3. .ait 빌드 관련
- `appName` 콘솔 등록값과 정확히 일치
- 빌드 한 번 한 후 코드 수정 → 재빌드 → 재업로드
- **로컬 deploymentId는 콘솔에 자동 등록되지 않음** → 콘솔에서 수동 업로드

### 4. 테스트 시
- `intoss-private://` URL은 휴대폰 토스 앱에서만 열림
- PC 브라우저에서 클릭 → "서비스 사용 불가" 에러

### 5. 검토 요청 시
- 한국어 기능명 다른 앱과 중복 X
- 영어 기능명 15자 이하
- 이동 URL의 `/screenName` 필수 입력

### 6. 출시 후
- 검색 즉시 노출 X (인덱싱 시간 필요)
- 새 버전 출시 시: 새 빌드 → 새 검토 (2~3일)
- 첫 사용자 데이터 적음 → 데이터 기반 결정은 2주 후
